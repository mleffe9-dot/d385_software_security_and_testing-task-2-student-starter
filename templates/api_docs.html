API Documentation

REST API for WGU Construction Equipment Rental System v2.0

Security Notice: This API is under security review. Several endpoints contain vulnerabilities that need to be addressed before production use.

Authentication

POST /api/v1/authenticate

Authenticate with username and password to receive an API key for subsequent requests.

Request Body
{ "username": "alice", "password": "alice456" }
Success Response (200)
{ "status": "success", "message": "Authentication successful", "api_key": "your_api_key_here" }

User Endpoints

GET /api/v1/user

Retrieve user information by username.

Query Parameters
Parameter Type Description
username string The username to look up
Example Request
GET /api/v1/user?username=alice
Security Issue: This endpoint is vulnerable to SQL injection and returns sensitive data including passwords!

Rental Endpoints

GET /api/v1/rentals

Retrieve all rental records. Optionally filter by username.

Query Parameters
Parameter Type Description
user string Optional - Filter rentals by username
Security Issue: No authentication required! Any user can view all rental records including sensitive SSN data.
POST /api/v1/rentals

Create a new equipment rental.

Request Body
{ "equipment": "Bulldozer", "days": 3, "username": "alice" }
Success Response (201)
{ "status": "success", "message": "Rental created", "data": { "id": 4, "user": "alice", "equipment": "Bulldozer", "days": 3, "total": 1500 } }
Security Issue: No input validation! Negative days and invalid equipment types are accepted.

Search

GET /api/v1/search

Search for equipment by name.

Query Parameters
Parameter Type Description
q string Search query
Example
GET /api/v1/search?q=Bull
Security Issue: Vulnerable to XSS - user input is reflected without sanitization!

Admin Endpoints

GET /api/v1/admin/users

Get a list of all users (admin only).

Critical Security Issue: This admin endpoint has NO authentication! Anyone can access it.
DELETE /api/v1/admin/delete_user

Delete a user account (admin only).

Query Parameters
Parameter Type Description
username string The username to delete
Critical Security Issue: Destructive operation with NO authentication! Anyone can delete users.

HTTP Response Codes

Code Status Description
200 OK Request successful - data returned in response body
201 Created Resource created successfully (e.g., new rental)
400 Bad Request Invalid input or malformed request
401 Unauthorized Authentication required but not provided
403 Forbidden Authenticated but insufficient permissions
404 Not Found Requested resource does not exist
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Unexpected server error occurred
